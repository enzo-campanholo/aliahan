# aliahan

Aliahan is a local app for planning external courses. The backend uses Gleam and the frontend uses Alpine.js. Its calendar can be generated by either the Gleam or SWI-Prolog scheduler.

It automatically lays out the modules so each course is finished before its deadline and to avoid overlaying modules on the same day. It also allows you to define a slack period before the deadline and rule out weekends. You can also reorder modules in the UI and do some color customization. The changes are persisted in a SQLite database in the root directory.

## Run it

Install Gleam, Erlang, SWI-Prolog, Node.js, and pnpm. Then build the checked-in frontend assets and start the server:

```sh
pnpm install --frozen-lockfile
pnpm run build:css
pnpm run vendor:alpine
gleam run
```

Open <http://127.0.0.1:8000>. You can add data in the UI or define `courses.toml` in the project root before the first run.

Set `ALIAHAN_DATABASE_PATH` to move the SQLite database. The TOML snapshot will follow it with a `.courses.toml` suffix. Set `ALIAHAN_COURSES_TOML_PATH` to choose the snapshot path directly.

## Check it

```sh
gleam format --check src test
gleam check
gleam test
swipl -q -g run_tests -t halt prolog/scheduler_tests.pl
pnpm run check:js
```

## Prolog scheduler

There is also a standalone SWI-Prolog scheduler in
`prolog/scheduler.pl`. It uses CLP(FD) to impose deadlines, module order,
and course prerequisites before search starts. Search then minimizes, in
order, the number of modules that share a day, the busiest day's load, and
the error from even spacing.

A course has the form
`course(Id, date(Year, Month, Day), PrerequisiteIds, RemainingModuleIds)`.
Use an empty module list for a completed course. Choose
`settings(weekends, SlackDays)` to allow every day, or
`settings(weekdays, SlackDays)` to skip Saturday and Sunday.

For example:

```prolog
?- use_module(prolog/scheduler).
?- Courses = [course(intro, date(2026,3,25), [], [i1]),
              course(next, date(2026,3,25), [intro], [n1,n2])],
   schedule(Courses, date(2026,3,19), settings(weekdays, 1), Entries).
```

Each result is an `entry(CourseId, ModuleId, Date, Slot)` term. Slot zero is
the first item on a day. `schedule/5` also yields
`score(Overlaps, PeakLoad, WorstSpacingError)`. `schedule/6` adds a list of
per-course `conflict(CourseId, Kind)` terms, omits impossible and blocked
courses, and still schedules the feasible courses. The shorter forms fail
when any course has a conflict.

The calendar toolbar has a **Gleam / Prolog** switch. The Prolog choice runs
`prolog/browser_scheduler.pl` through `swipl`, converts its result to the same
JSON shape as the Gleam schedule, and previews it without overwriting the
stored Gleam schedule. The choice is kept in browser storage.

If `swipl` is not on `PATH`, set `ALIAHAN_SWIPL_PATH` to its executable. Set
`ALIAHAN_PROLOG_RUNNER_PATH` if the app starts outside the repository root and
cannot find the browser scheduler script.
